## 📌 ADHD CBT 앱 4개 템플릿 제안

### **1. 설명 템플릿 (교육·정보 제공 화면)**

**목적:** 사용자가 훈련의 필요성·효과·이론적 배경을 충분히 이해하도록 함.
**구성:**

* **상단:** 훈련 제목 + 진행 Day 표시 ("Day 1 / 3")
* **메인 영역:**

  * 이미지/일러스트 (상황 설명, 사례 시각화)
  * **본문 텍스트(스크롤 가능)**

    * 훈련의 필요성 (왜 해야 하는지)
    * 이점·효과 (ADHD 증상 개선 사례, 논문 결과)
    * 실제 사례·스토리텔링 (이입 유도)
    * 오늘 진행 목표 (예: "오늘은 물건 잃어버림을 줄이기 위한 인식 훈련을 배웁니다.")
* **하단 버튼:** "다음 →" (다음 화면 또는 목록 복귀)
* **UI 요소:**

  * **상단 진행바** (훈련 전체 중 현재 위치 표시)
  * 가독성 높은 서체와 적절한 줄간격
  * 주요 문장 하이라이트 기능(형광펜 스타일)

---

### **2. 문제풀이 템플릿 (퀴즈형 실습)**

**목적:** 사용자가 설명 내용을 이해했는지 확인하고, 실제 적용 아이디어를 떠올리게 함.
**구성:**

* **상단:** "문제 1 / 3" 표시
* **문제 유형:**

  * 객관식 4지선다 (예: 올바른 전략 선택)
  * 상황형 질문 + 선택지 (예: "회의 중 집중이 흐트러졌을 때 가장 좋은 대처는?")
  * 드래그 매칭(선택한 전략을 해당 상황에 연결)
* **피드백:**

  * 정답 선택 시 "👍 맞아요! 이 전략은 연구에서 집중력을 20% 향상시켰습니다."
  * 오답 선택 시 "❌ 이런 방법도 가능하지만, ADHD에 더 효과적인 방법은 ○○입니다."
* **하단 버튼:** "다음 문제 →"
* **UI 요소:**

  * 문제당 남은 시간 표시(선택 사항)
  * 간단한 애니메이션으로 피드백 제공

---

### **3. 기록 템플릿 (설문·자기 보고)**

**목적:** 훈련 후 경험·느낌·난이도를 기록해 패턴을 분석하고, 개인화 피드백 제공
**구성:**

* **상단:** 훈련명 + "기록" 표시
* **설문 문항:**

  1. 오늘 훈련의 난이도는? (매우 쉬움\~매우 어려움, 5단계 척도)
  2. 만족도는? (5단계 척도)
  3. 오늘 훈련을 하면서 가장 어려웠던 점은? (객관식/간단 주관식)
  4. 훈련 적용 가능성은? (5단계 척도)
  5. 레슨별 추가 질문 1개 (예: "오늘 물건 위치를 의식적으로 기억하려 노력했나요?")
* **하단 버튼:** "기록 제출"
* **UI 요소:**

  * 슬라이더·라디오버튼 기반의 직관적 선택 방식
  * 입력 즉시 저장(자동 저장)

---

### **4. 피드백 템플릿 (맞춤형 종합 결과)**

**목적:** 훈련 전체를 마친 후, 문제풀이·기록 데이터 기반으로 개인화된 행동 지침 제공
**구성:**

* **상단:** "훈련 결과 요약"
* **메인 영역:**

  * **점수 요약:** 문제풀이 정답률, 평균 난이도·만족도
  * **성취 분석:** 오늘/주간 비교 그래프
  * **개인화 피드백:**

    * 강점: "당신은 시각적 단서를 활용하는 전략을 잘 적용하고 있습니다."
    * 개선점: "물건을 둘 때 속도를 늦추는 습관을 추가하면 효과가 더 커집니다."
    * 추천 행동: "다음 주에는 AirTag와 같은 물리적 도구를 함께 사용해보세요."
  * **추가 학습 제안:** 관련 훈련 연결(다음 레슨 추천)
* **하단 버튼:** "다음 훈련 시작"
* **UI 요소:**

  * 시각화 그래프(막대/도넛형)
  * 컬러 코드(녹색=성취, 노랑=보통, 빨강=개선 필요)

---

📌 **핵심 설계 포인트**

* 각 훈련은 **"설명 → 문제풀이 → 기록 → 피드백"** 중 필요한 단계만 사용 가능
* 난이도·일수에 따라 **문제풀이/기록 반복** 가능
* 피드백은 **마지막 1회만 표시**
* UI 일관성 유지로 사용자 혼란 최소화
